## 學習重點

- **動機**：與其每天自己一支一支看盤，不如下一個指令，讓 AI 定時抓資料、整理、回傳。
- **資料源頭最重要**：問 ChatGPT/Gemini「現在股價」，約 1/3～1/2 會出錯——因為它是上 Google 找的**二三手、可能是前一天**的資料。要的是「源頭」（券商／API／即時網站），而且用**程式**去抓。
- **程式碼是 AI 的母語**：把任務變成可跑的 Python，精準度幾乎 99.99%（確定性），勝過用自然語言問 AI（會誤解）。**Python 存下來 ＝ 隨時可重跑、換任何 AI／環境都能跑、不必綁訂閱**。
- **AI agent 的本事**：一條路被擋（404／防爬）會自動換另一條（玩股網 → Yahoo → Nasdaq）直到拿到正確資料；錯了會自己 debug 重試，不像舊 AI 一錯就停。
- **工作流三站**：① 抓取 → 清洗 → 結構化（＝資料清洗，導入 AI 的第一步）② 判斷邏輯（你的操作策略：KD>80／<20、量 > 5 日均 15%）③ 符合條件 → highlight → 用 LINE／Telegram 通知。

## 討論內容與問答

### 為什麼 AI 報的股價常常錯
資料流是關鍵：AI 上 Google 找 = 二三手，可能抓到前一天。最直接的即時股價要從券商軟體 / API / 即時網站來。提示工程上：別只說「給我這 10 隻股價」，要說「到 [玩股網／Yahoo] 抓這 10 隻的即時價」——但更穩的是用程式直接抓源頭。

### 用 Codex 抓即時股價（實作）
- 鎖定網站先測「會不會防爬」：兩大陣營——擁抱 AI 爬取 vs 只給人看（為了廣告、賣課）。
- 標的：MU（美光）、GOOGL（Google）、SpaceX（剛 IPO，可能還沒資料）等。
- Prompt 很短：「從這個網站抓即時股價 [清單]，建立定時監控、觀察走勢」。
- **對帳發現的坑**：Codex 報 6/12 為 995，但網站顯示 6/12 是 981 → 是**玩股網自己的渲染 bug** ＋ **美股用美國時間（與台灣差一天）**。需另開網站／手機 App 核對。
- Codex 在**沙盒（虛擬機）**裡寫 Python、執行、要權限時按 `4`（或 always-4 不再問）；跨多次嘗試自動 debug。模型為 GPT-5.x（夠聰明會自己換源）。
- **公開資料不需要 API key**：有一版程式誤用了需付費 key，改成抓公開資料的免 key 版即可。

### 這支 Python 在哪裡跑
- **Google Colab（免費雲端虛擬機）最省事**：貼上程式 → 按 run；出錯就把錯誤碼貼回 Codex → 拿修正版貼回 Colab 重跑。不必自己裝、養機器。
- 或本機虛擬機（約半小時裝起來）。
- 未來：本地小模型機（NVIDIA Spark／「龍蝦雞」十幾萬、夠算力、可跑堪用小模型）。

### 硬體評估
- Mac 最適合 AI（尤其搭 iPhone）；但若要 Windows 專用軟體（ERP／交易程式）才需 Windows。
- 趨勢：**簡單／明確任務交本地小模型**（免費、如跑 Python），**大規劃／寫程式才連雲端大腦**設計流程，平常用地端小模型跑。

### 其他工具
- **Perplexity**：直接就把抓取程式寫出來；財務／健康／學術／專利特別強。
- **Comet（Perplexity 瀏覽器）**：操控鍵鼠、像人一樣繞過防爬。
- **NotebookLM**：現在可直接吃新的 Excel 格式，不用先轉檔。

## 踩過的坑

- 問 AI「現在股價」→ 可能給前一天／錯的 → 指定源頭並用程式抓。
- 玩股網渲染 bug ＋ 美股美國時間（差一天）→ 對帳要另開網站／手機核對。
- 防爬網站抓不到 → 讓 Codex 自動換源（Yahoo US／Nasdaq）。
- 產出程式要 API key（付費）→ 抓公開資料其實不需要 → 請它改免 key 版。
- Colab 跑出錯 → 錯誤碼貼回 Codex → 修正版貼回 Colab 重跑。
